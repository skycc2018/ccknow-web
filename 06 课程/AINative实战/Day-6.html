🎓 AI Native 实战课程 - Day 6: 第六章 - 工具调用与 Agent

📖 今日导读

在 [[AI Native 深度解析：从概念到落地]] 中，[[《AI Native：从概念到落地》- 书籍规划]] 把第六章命名为「工具调用与 Agent」。如果说前几章解决的是 AI “怎么想、怎么说”，那么今天解决的是 AI “怎么做事”。

大模型本身只有“截止到训练数据的知识”，无法访问实时股价、无法操作你的日历、无法读取企业私有数据库。而 工具调用（Function Calling）让模型获得了“伸出双手”的能力：它可以调用搜索引擎、查询天气、发送邮件、执行代码、甚至操作浏览器。当工具调用与推理能力结合，就诞生了 Agent——一个能够自主感知、决策、执行的智能体。

今天的核心认知是：AI Native 产品的最终形态不是“更好的聊天机器人”，而是“能独立完成任务的数字员工”。2025–2026 年，从 Anthropic 的企业调研到 Manus、Cursor、Claude Code 的现象级产品，再到 MCP 协议的快速普及，整个行业都在验证同一件事：Agent 正在从实验走向生产。

🔍 核心概念

1. 工具调用：让 AI 从“说话”到“办事”

工具调用 是指大语言模型根据用户意图，主动选择并调用外部函数/API 的能力。它不是简单的字符串拼接，而是一个结构化的“决策-执行-反馈”过程：

  1. 模型判断需要外部信息或操作；
  2. 模型输出 JSON 格式的函数调用请求（函数名 + 参数）；
  3. 应用层执行函数，获得结果；
  4. 结果返回给模型，模型继续推理或生成最终回答。

例如：用户问“北京今天需要带伞吗？”AI 不会凭训练数据瞎猜，而是调用 get_weather(city="北京")，拿到实时天气后再回答。

2. Agent：具备自主权的 AI 系统

Agent（智能体）是一个能够在环境中感知、推理并采取行动以实现目标的系统。与传统 AI 功能相比，Agent 的核心特征是：

维度 传统 AI 功能 Agent
交互方式 单次输入 → 单次输出 多轮思考 → 行动 → 观察
与外部世界 无连接 通过工具调用连接系统
自主性 被动响应 主动规划、执行、纠错
任务复杂度 单一、明确 多步骤、可分解

3. ReAct：思考-行动-观察循环

ReAct（Reasoning + Acting）是构建 Agent 最经典的心智模型：

用户问题
  ↓
Thought（思考）：我需要什么信息/操作？
  ↓
Action（行动）：调用某个工具
  ↓
Observation（观察）：工具返回了什么？
  ↓
（循环，直到可以给出 Final Answer）
  ↓
Final Answer（最终答案）

ReAct 把“推理链”和“工具执行”结合起来，让 Agent 既能像人一样思考，又能像软件一样操作外部系统。它是 LangChain、LangGraph、AutoGen 等框架设计 Agent 的底层范式。

4. MCP：AI 工具的“USB-C”接口

2024 年底由 Anthropic 推出的 MCP（Model Context Protocol，模型上下文协议）正在快速成为行业事实标准。它的核心思想是：

与其为每个大模型/每个数据源单独写适配器，不如定义一个统一协议，让任何 AI 应用都能“即插即用”地访问任何工具或数据源。

MCP 把外部能力抽象为 Server（提供资源/工具）和 Client（消费这些能力的 AI 应用）。截至 2025–2026 年，GitHub、Slack、数据库、浏览器、文件系统等都已经出现大量 MCP Server，形成了一个蓬勃的插件生态。

5. Multi-Agent：模拟人类团队协作

复杂任务往往无法由单一 Agent 完成。Multi-Agent 系统把任务拆分给多个专业 Agent：

  • 规划 Agent：拆解任务、分配子目标；
  • 执行 Agent：调用工具完成具体动作；
  • 审核 Agent：检查事实、安全、合规；
  • 总结 Agent：整合多源输出，生成最终交付物。

微软 AutoGen 和 CrewAI 是这一方向的代表性框架。Multi-Agent 不是“多开几个 ChatGPT”，而是让不同角色的 Agent 通过定义好的协作协议完成复杂工作流。

💻 实战代码

下面的脚本演示了一个最小可运行的 ReAct Agent。它的核心思想是：

  1. 定义若干工具（计算器、搜索、天气、当前时间）；
  2. 用规则驱动的 Mock LLM 模拟“思考 → 行动 → 观察”的 ReAct 循环；
  3. 展示 Agent 如何根据用户问题自动选择合适的工具；
  4. 代码中预留了 call_llm() 替换点，接入 OpenAI/Anthropic 等真实模型即可用于生产。
"""
Day 6 实战代码：工具调用与 ReAct Agent
一个最小可运行的 ReAct Agent。无需 API Key，内置 Mock LLM。
如需接入真实模型，替换 mock_llm() 函数即可。

运行方式：
  python3 day6_code.py
"""
import json
import re
from datetime import datetime
from typing import Dict, List, Any, Callable


# ------------------- 工具定义 -------------------
class Tool:
    """Agent 可调用的外部工具。"""

    def __init__(
        self,
        name: str,
        description: str,
        parameters: Dict[str, Any],
        func: Callable,
    ):
        self.name = name
        self.description = description
        self.parameters = parameters
        self.func = func

    def schema(self) -> Dict[str, Any]:
        return {
            "type": "function",
            "function": {
                "name": self.name,
                "description": self.description,
                "parameters": self.parameters,
            },
        }

    def call(self, args: Dict[str, Any]) -> str:
        try:
            return str(self.func(**args))
        except Exception as e:
            return f"Error: {e}"


# ------------------- 具体工具实现 -------------------
def calculator(expression: str) -> str:
    """安全计算数学表达式（仅允许数字和四则运算符）。"""
    allowed = set("0123456789+-*/(). ")
    if not all(c in allowed for c in expression):
        return "Error: 仅支持基础四则运算"
    try:
        return f"{eval(expression)}"
    except Exception as e:
        return f"Error: {e}"


def search(query: str) -> str:
    """Mock 搜索引擎：在本地知识库中匹配关键词。"""
    knowledge = {
        "李白": "李白（701-762），字太白，号青莲居士，唐代伟大浪漫主义诗人，被后人誉为“诗仙”。",
        "北京奥运会": "2008 年夏季奥林匹克运动会在北京举办，是第 29 届夏季奥运会。",
        "MCP": "MCP（Model Context Protocol）是 Anthropic 提出的模型上下文协议，让 AI 应用能统一接入外部工具和数据源。",
    }
    for k, v in knowledge.items():
        if k in query:
            return v
    return f"Mock 搜索结果：找到关于「{query}」的若干相关信息（示例）。"


def get_weather(city: str) -> str:
    """Mock 天气查询。"""
    mock = {
        "北京": "晴，25°C，空气质量良",
        "上海": "小雨，22°C，湿度 78%",
        "深圳": "多云，28°C，适宜出行",
    }
    return f"{city}天气：{mock.get(city, '未知，请查询其他城市')}。"


def get_date() -> str:
    """获取当前日期时间。"""
    return datetime.now().strftime("%Y-%m-%d %H:%M:%S")


# 工具注册表
TOOLS: List[Tool] = [
    Tool(
        name="calculator",
        description="计算数学表达式，例如 '2 + 3 * 4' 或 '(100 - 25) / 3'",
        parameters={
            "type": "object",
            "properties": {
                "expression": {
                    "type": "string",
                    "description": "要计算的数学表达式",
                }
            },
            "required": ["expression"],
        },
        func=calculator,
    ),
    Tool(
        name="search",
        description="搜索事实性知识，例如历史人物、事件、概念等",
        parameters={
            "type": "object",
            "properties": {
                "query": {
                    "type": "string",
                    "description": "搜索关键词或问题",
                }
            },
            "required": ["query"],
        },
        func=search,
    ),
    Tool(
        name="get_weather",
        description="查询指定城市的天气",
        parameters={
            "type": "object",
            "properties": {
                "city": {
                    "type": "string",
                    "description": "城市名，例如 北京、上海、深圳",
                }
            },
            "required": ["city"],
        },
        func=get_weather,
    ),
    Tool(
        name="get_date",
        description="获取当前日期时间",
        parameters={"type": "object", "properties": {}},
        func=get_date,
    ),
]

TOOLS_MAP = {t.name: t for t in TOOLS}


# ------------------- Mock LLM -------------------
def build_prompt(query: str, history: List[Dict[str, str]]) -> str:
    """构造 ReAct 格式的 Prompt。"""
    tools_desc = json.dumps([t.schema() for t in TOOLS], ensure_ascii=False, indent=2)
    prompt = f"""你是一个乐于助人的 AI Agent。你可以使用以下工具：
{tools_desc}

请严格按以下格式回复，每轮只输出一个 Thought + Action 或 Thought + Final Answer：
Thought: <你的思考过程>
Action: <工具名>
Action Input: <JSON 格式的参数>

当可以回答用户时，使用：
Thought: <你的思考过程>
Final Answer: <最终答案>

当前问题：{query}

历史记录：
"""
    for h in history:
        prompt += f"{h['role']}: {h['content']}\n"
    prompt += "Assistant: "
    return prompt


def mock_llm(prompt: str) -> str:
    """
    基于规则的 Mock LLM，仅用于演示 ReAct 循环。
    生产环境请替换为 OpenAI/Anthropic/Claude 等真实 API 调用。
    """
    if "李白" in prompt:
        if "search" not in prompt.lower():
            return (
                'Thought: 用户询问李白是谁，我需要搜索相关知识。\n'
                'Action: search\n'
                'Action Input: {"query": "李白"}'
            )
        return (
            'Thought: 我已经获得李白的相关信息，现在可以给出最终答案。\n'
            'Final Answer: 李白（701-762），字太白，号青莲居士，唐代伟大浪漫主义诗人，被后人誉为“诗仙”。'
        )

    if "北京" in prompt and "天气" in prompt:
        if "get_weather" not in prompt.lower():
            return (
                'Thought: 用户询问北京天气，我应该调用天气工具获取实时信息。\n'
                'Action: get_weather\n'
                'Action Input: {"city": "北京"}'
            )
        return (
            'Thought: 已经获取北京天气信息，现在给出最终答案。\n'
            'Final Answer: 北京今天天气晴，25°C，空气质量良，适合出行。'
        )

    if "25" in prompt and "37" in prompt:
        if "calculator" not in prompt.lower():
            return (
                'Thought: 这是一个数学计算问题，我需要使用计算器。\n'
                'Action: calculator\n'
                'Action Input: {"expression": "25 + 37"}'
            )
        return (
            'Thought: 计算已完成，现在给出最终答案。\n'
            'Final Answer: 25 + 37 = 62。'
        )

    return (
        'Thought: 这是一个通用问题，我直接回答。\n'
        'Final Answer: 我是 ReAct Agent 演示，请尝试问我“李白是谁”、“北京今天天气怎么样”或“25+37等于多少”。'
    )


def parse_action(response: str) -> Dict[str, Any]:
    """解析 LLM 输出：判断是工具调用还是最终答案。"""
    action_match = re.search(
        r"Action:\s*(\w+)\s*Action Input:\s*(\{.*?\})",
        response,
        re.S,
    )
    if action_match:
        return {
            "type": "action",
            "name": action_match.group(1),
            "args": json.loads(action_match.group(2)),
        }

    final_match = re.search(r"Final Answer:\s*(.*)", response, re.S)
    if final_match:
        return {"type": "final", "answer": final_match.group(1).strip()}

    return {"type": "unknown", "raw": response}


def run_react(query: str, max_steps: int = 5) -> str:
    """执行 ReAct 循环，直到获得最终答案或达到最大步数。"""
    history = []
    print(f"用户问题：{query}")

    for i in range(max_steps):
        prompt = build_prompt(query, history)
        response = mock_llm(prompt)
        print(f"\nStep {i + 1}:\n{response}")

        parsed = parse_action(response)
        if parsed["type"] == "final":
            return parsed["answer"]

        if parsed["type"] == "action":
            tool = TOOLS_MAP.get(parsed["name"])
            if not tool:
                observation = f"Error: 未知工具 {parsed['name']}"
            else:
                observation = tool.call(parsed["args"])
            print(f"Observation: {observation}")
            history.append({"role": "Assistant", "content": response})
            history.append({"role": "Observation", "content": observation})
            continue

        return "无法解析 Agent 响应。"

    return "达到最大步数，未获得最终答案。"


if __name__ == "__main__":
    queries = [
        "李白是谁？",
        "北京今天天气怎么样？",
        "25 + 37 等于多少？",
    ]
    for q in queries:
        print("=" * 60)
        answer = run_react(q)
        print(f"\n最终答案：{answer}\n")

代码启示：Agent 的核心不是“更多 Prompt”，而是 “思考-行动-观察”的闭环。只要工具定义清晰、边界可控，Agent 就能在复杂任务中自主推理。

✅ 最佳实践

结合 Anthropic 2026 企业调研、MCP 社区实践以及主流 Agent 框架的工程经验，总结出 8 条可落地的设计指南：

  1. 工具定义要“自解释” 函数名、参数名、description 都要让模型能看懂。避免模糊命名，参数类型和必填项必须明确。模型只能看到你给的 schema，看不到实现细节。

  2. Prompt 里保留 ReAct 骨架 不要让模型自由发挥。显式要求模型输出 Thought / Action / Action Input / Final Answer，并给出格式示例，能显著提升调用准确率。

  3. 每个工具都要有“输入校验”和“错误回退” Agent 可能会传错参数。工具内部要做参数校验、异常捕获，并返回清晰的错误信息给模型，让它能自我修复。

  4. 限制循环深度和工具调用次数 再聪明的 Agent 也可能陷入死循环。设置 max_steps、单次调用超时、总 Token 上限，是生产部署的底线。

  5. 把 Agent 当作核心基础设施，而不是实验项目 Anthropic 的调研显示，把 Agent 纳入日常工程/业务流程的团队更容易获得回报。这意味着要有 CI/CD、监控、版本管理和成本核算。

  6. 优先解决“集成”和“数据”问题 企业部署 Agent 的最大挑战是：与现有系统集成（46%）、数据访问与质量（42%）。产品设计上要为 Agent 预留标准 API、统一权限和高质量数据接口。

  7. 人机协作不可省略 关键操作（发送邮件、转账、发布内容、修改生产配置）必须设置人工确认节点。Agent 负责“ draft + 推荐”，人类负责“最终决策”。

  8. 从编码 Agent 开始，逐步扩展到其他场景 代码生成是 Agent 最成熟的落地场景（90% 企业用于开发辅助）。先在一个高 ROI、可量化、容错性好的场景跑通，再向客服、研究、运营等场景复制。
