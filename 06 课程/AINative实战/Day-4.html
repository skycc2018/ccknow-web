🎓 AI Native 实战课程 - Day 4: 第四章 - 上下文即状态

📖 今日导读

[[《AI Native：从概念到落地》- 书籍规划]] 把第四章命名为「上下文即状态」，这是整本书承上启下的一章。第三章讲 Prompt 即接口，解决的是「AI 如何理解单次指令」；第四章要解决的是「AI 如何理解一段关系」—— 它必须记得你是谁、你们之前聊过什么、你偏好什么，否则永远只是一个用完即走的工具。

今天的核心认知是：大模型本身是无状态的，每一次 API 调用都是独立的。 所谓「记忆」，不是模型学会了什么，而是我们把外部状态（用户画像、会话历史、任务进度、业务数据）注入到下一次请求的上下文里。上下文即状态，状态即服务。本章结合 2025 年 ChatGPT、Claude 等产品的最新记忆能力，以及开源社区涌现的记忆基础设施，给你一个可运行的「有状态 AI 助手」示例。

🔍 核心概念

1. 大模型是无状态的，记忆是工程问题

在 [[AI Native 深度解析：从概念到落地]] 中，「记忆与上下文」被列为五大核心原理之一。LLM 的每一次推理都只依赖当前输入的 token；它不会自动记住你三分钟前说过的话，更不会记得上周的讨论。所谓「AI 记得你」，本质上是产品层做了一件工程化的事：把需要保留的信息挑选出来，保存到某种外部存储，然后在下一次请求时重新注入上下文。

因此，「上下文即状态」不是修辞，而是一个架构命题：

  • 状态 = 任何会影响下一次决策的信息
  • 上下文 = 当前被注入到模型输入中的那部分状态
  • 上下文工程 = 决定哪些状态该留、该丢、该压缩、该检索

2. 三层上下文状态

层级 名称 典型内容 存储介质 生命周期
L1 工作上下文 当前 prompt、最近几轮对话、系统提示词 内存 / 请求体 单次请求
L2 会话状态 多轮任务进度、工具调用结果、临时变量 Redis / 内存 一次会话
L3 持久记忆 用户偏好、关键事实、业务知识、历史摘要 数据库 / 向量库 跨会话、跨设备

大多数 AI Native 产品失败，不是因为模型不够强，而是因为 L2 和 L3 设计得太随意：会话一刷新就丢失进度，用户重复说同一句话，或者把不相关的历史全部塞进上下文导致模型「失忆」。

3. 从「记住一切」到「召回正确的」

上下文窗口虽然越来越长（Gemini 2.5 达 100 万 token，Llama 4 达 1000 万 token），但「能塞进去」不等于「能用得好」。Hazy Research 2025 年 6 月的研究指出：在单张 H100 上，如果用户上下文从 1024 token 增到 128k token，Llama 8B 的服务吞吐会从约 10,000 token/s 暴跌到约 130 token/s，下降 77 倍。这背后的核心瓶颈就是 KV Cache。

所以行业趋势不是无脑堆长上下文，而是：

  • RAG / 语义检索：只在需要时召回相关片段
  • 记忆摘要：把长对话压缩成结构化摘要
  • KV Cache 压缩：如 Hazy Research 的 Cartridges 通过「self-study」把长上下文压缩成小型 cartridge， reportedly 提升峰值吞吐 26 倍
  • Prompt Caching：缓存重复的系统提示词和上下文，降低推理成本

4. AI Native 的状态设计思维

把上下文当状态，意味着产品架构要从「请求-响应」升级为「状态机」：

  • 状态要有 schema：用户画像、项目记忆、任务状态、环境变量，不能只靠一个巨大的 prompt 字符串
  • 状态要可观测：用户能查看、编辑、删除 AI 记住了什么
  • 状态要有边界：项目之间、租户之间、会话之间不能串味
  • 状态要能迁移：用户换设备、换模型时，记忆可以导出/导入

这也是 2025 年头部产品纷纷推出「记忆面板」「隐私聊天」「项目隔离」的原因 —— 用户开始意识到：谁控制记忆，谁就控制 AI 的「人格」。

💼 最新案例

案例一：ChatGPT Memory —— 从「保存的记忆」到「全聊天记录」

2025 年 4 月 10 日，OpenAI 宣布 ChatGPT 的记忆能力重大升级：Plus / Pro 用户的 AI 可以引用所有历史对话来生成更个性化的回答。这比之前的「Saved memories」走得更远 —— 后者需要用户手动告诉 AI「请记住这个」，而新版本会自动从过往聊天中推断偏好和兴趣。

  • 数据与节奏：4 月先向 Plus / Pro 开放；6 月 3 日开始向免费用户逐步推出「Chat history」记忆
  • 产品逻辑：ChatGPT 会维护一份关于用户的 detailed summary，并在每次生成时作为系统提示词的一部分注入
  • 争议：Simon Willison 在 2025 年 5 月的博客 I really don’t like ChatGPT’s new memory dossier 中指出，这种隐性记忆会不可预测地影响输出（例如他让 ChatGPT 根据照片猜地点，结果 AI 从过往聊天中知道了他住在 Half Moon Bay）。这引出一个关键设计问题：个性化与可控性之间的张力

启示：上下文即状态的设计必须回答两个问题 —— 用户是否知道 AI 记住了什么？用户是否能轻松编辑或关闭它？

案例二：Claude Memory —— 项目隔离 + 可控记忆

2025 年 10 月 23 日，Anthropic 发布 Claude Memory，面向 Team / Enterprise / Pro / Max 用户。与 ChatGPT 的「全历史记忆」不同，Claude 的设计更强调 工作场景和边界控制：

  • 项目级记忆：每个 Project 有独立的 memory，销售线索、产品需求、客户背景不会混到个人聊天里
  • 记忆摘要可编辑：用户可以在设置里看到 Claude 记住了什么，并直接修改 summary
  • 隐私模式：Incognito chat 不会进入历史记录，也不会写入 memory
  • 安全测试：Anthropic 明确提到在上线前针对 wellbeing、过度迎合、绕过安全护栏等场景做了大量测试

启示：AI Native 的 memory 不是「越多越好」，而是「边界越清晰越安全」。项目隔离、用户可控、隐私模式，是记忆设计的三大基准。

案例三：Claude Code 的「episodic memory」—— 程序员的上下文修复

2025 年 10 月，开发者 Jesse 在博客 Fixing Claude Code's amnesia 中分享了一个现象：Claude Code 虽然保存了每次会话日志，但默认不会主动检索。于是他给 Claude Code 写了一个 episodic memory 插件：

  • 自动归档历史会话到本地 SQLite + 向量索引
  • 通过 MCP 工具让 Claude 可以语义搜索「我们上次是怎么解决这个问题的」
  • 用一个单独的 compression prompt 管理上下文膨胀

经过 3–4 周的 dogfooding，他发现 Claude 终于能记得「昨天在做什么」「上次 rejected 的方案是什么」。这验证了 AI 编程伙伴的下一个竞争点：不是代码生成速度，而是跨会话的上下文连续性。

案例四（开源项目）：Letta / Memori / Honcho —— 记忆基础设施

如果说 2025 年前大家都在自己写记忆层，2025–2026 年则出现了专门的记忆基础设施：

  • Letta (letta-ai/letta, 23.5k ⭐)：定位「stateful agents」平台，强调 AI 能通过高级记忆持续学习和自我改进，最新版 v0.16.8 于 2026 年 5 月发布
  • Memori (MemoriLabs/Memori, 15.4k ⭐)：agent-native memory infrastructure，把 agent 执行和对话转化为结构化、持久的 state，支持 memory MCP
  • Honcho (plastic-labs/honcho, 5.5k ⭐)：开源 memory library，用 custom models 做长期记忆，强调 personalization 和 continual learning

启示：记忆正在从「每个产品自己造轮子」变成「可复用的基础设施层」。这和数据库当年的演进非常相似。

💻 实战代码

下面的 Python 脚本演示了上下文即状态的最小可行实现：一个带「短期记忆（对话历史）」「长期记忆（用户画像）」「状态持久化（本地 JSON）」的 AI 助手。

"""
Day 4 实战代码：上下文即状态
目标：演示一个带短期记忆、长期记忆和状态持久化的 AI 助手。

运行方式：
  1. 不设置 OPENAI_API_KEY：使用本地规则生成回复，可立即运行
  2. 设置 OPENAI_API_KEY：调用 GPT-4o-mini，把记忆注入系统提示词

依赖：pip install openai（可选）
"""

import os
import json
import re
from dataclasses import dataclass, field, asdict
from datetime import datetime
from typing import List, Dict

STATE_FILE = "assistant_state.json"
MAX_WORKING_MEMORY = 6  # 保留最近 3 轮对话（每轮 user + assistant = 2 条）


@dataclass
class UserProfile:
    """长期记忆：用户画像。"""
    preferences: Dict[str, str] = field(default_factory=dict)
    facts: List[str] = field(default_factory=list)
    summary: str = ""


@dataclass
class AssistantState:
    """会话状态：短期记忆 + 长期记忆。"""
    user_id: str = "default"
    working_memory: List[Dict[str, str]] = field(default_factory=list)
    profile: UserProfile = field(default_factory=UserProfile)
    updated_at: str = field(default_factory=lambda: datetime.now().isoformat())

    def to_dict(self) -> dict:
        return {
            "user_id": self.user_id,
            "working_memory": self.working_memory,
            "profile": asdict(self.profile),
            "updated_at": self.updated_at,
        }

    @classmethod
    def from_dict(cls, data: dict) -> "AssistantState":
        profile = UserProfile(**data.get("profile", {}))
        return cls(
            user_id=data.get("user_id", "default"),
            working_memory=data.get("working_memory", []),
            profile=profile,
            updated_at=data.get("updated_at", datetime.now().isoformat()),
        )


class StatefulAssistant:
    """
    上下文即状态的简单实现：
      - 把对话历史当作短期记忆
      - 把用户画像当作长期记忆
      - 把状态持久化到本地 JSON
    """

    def __init__(self, user_id: str = "default", state_file: str = STATE_FILE):
        self.user_id = user_id
        self.state_file = state_file
        self.state = self._load_state()

    # ------------------- 状态持久化 -------------------
    def _load_state(self) -> AssistantState:
        if os.path.exists(self.state_file):
            with open(self.state_file, "r", encoding="utf-8") as f:
                return AssistantState.from_dict(json.load(f))
        return AssistantState(user_id=self.user_id)

    def save_state(self):
        self.state.updated_at = datetime.now().isoformat()
        with open(self.state_file, "w", encoding="utf-8") as f:
            json.dump(self.state.to_dict(), f, ensure_ascii=False, indent=2)

    # ------------------- 短期记忆管理 -------------------
    def add_exchange(self, user_input: str, assistant_reply: str):
        self.state.working_memory.append({"role": "user", "content": user_input})
        self.state.working_memory.append({"role": "assistant", "content": assistant_reply})
        # 只保留最近的 MAX_WORKING_MEMORY 条消息
        if len(self.state.working_memory) > MAX_WORKING_MEMORY:
            self.state.working_memory = self.state.working_memory[-MAX_WORKING_MEMORY:]

    # ------------------- 长期记忆更新 -------------------
    def update_long_term_memory(self, user_input: str):
        """用简单规则从用户输入中提取偏好和事实。"""
        text = user_input

        # 提取偏好："我喜欢..." / "请用..."
        pref_patterns = [
            r"我(?:喜欢|偏好|习惯)([^。，什么谁怎么如何]+)",
            r"请用([^。，什么谁怎么如何]+?)(?:回复|回答|输出)",
            r"(?:回答|回复|输出)要([^。，什么谁怎么如何]+)",
        ]
        for pat in pref_patterns:
            m = re.search(pat, text)
            if m:
                value = m.group(1).strip()
                if value:
                    self.state.profile.preferences["style"] = value

        # 提取事实："我在...工作" / "我是..." / "记住..."
        fact_patterns = [
            r"记住([^。，什么谁怎么如何]+)",
            r"我是([^。，什么谁怎么如何]+)",
            r"我在([^。，什么谁怎么如何]+)",
            r"我从事([^。，什么谁怎么如何]+)",
        ]
        for pat in fact_patterns:
            for m in re.finditer(pat, text):
                fact = m.group(1).strip()
                if fact and fact not in self.state.profile.facts:
                    self.state.profile.facts.append(fact)

        # 更新摘要
        if self.state.profile.facts or self.state.profile.preferences:
            parts = []
            if self.state.profile.preferences:
                parts.append(
                    "用户偏好：" + "；".join(f"{k}={v}" for k, v in self.state.profile.preferences.items())
                )
            if self.state.profile.facts:
                parts.append("已知事实：" + "；".join(self.state.profile.facts))
            self.state.profile.summary = "；".join(parts)

    # ------------------- 记忆检索（简化版 RAG） -------------------
    def recall_relevant_memory(self, user_input: str) -> str:
        """根据关键词从长期记忆中检索相关内容。"""
        keywords = set(re.findall(r"\w+", user_input.lower()))
        matches = []
        for fact in self.state.profile.facts:
            if keywords & set(re.findall(r"\w+", fact.lower())):
                matches.append(fact)
        if matches:
            return "相关记忆：" + "；".join(matches)
        return ""

    # ------------------- 回复生成 -------------------
    def build_context(self, user_input: str) -> str:
        context_parts = [
            "你是一位 AI 学习助手。请根据下面的上下文状态回答用户问题。",
            "",
        ]
        if self.state.profile.summary:
            context_parts.append(f"【长期记忆】{self.state.profile.summary}")
            context_parts.append("")

        relevant = self.recall_relevant_memory(user_input)
        if relevant:
            context_parts.append(f"【本次召回】{relevant}")
            context_parts.append("")

        if self.state.working_memory:
            context_parts.append("【近期对话】")
            for msg in self.state.working_memory:
                role = "用户" if msg["role"] == "user" else "助手"
                context_parts.append(f"{role}：{msg['content']}")
            context_parts.append("")

        context_parts.append(f"用户当前输入：{user_input}")
        return "\n".join(context_parts)

    def _reply_with_llm(self, user_input: str) -> str:
        from openai import OpenAI

        client = OpenAI()
        response = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[
                {"role": "system", "content": self.build_context(user_input)},
                {"role": "user", "content": user_input},
            ],
            temperature=0.7,
        )
        return response.choices[0].message.content

    def _reply_local(self, user_input: str) -> str:
        """无 API Key 时的本地规则回复，仅用于演示状态如何使用。"""
        text = user_input.lower()

        # 使用已记住的偏好
        style = self.state.profile.preferences.get("style", "")
        style_hint = f"（我会尽量{style}）" if style else ""

        # 使用已记住的事实
        if "我是谁" in user_input or "我叫什么" in user_input:
            if self.state.profile.facts:
                return f"根据我的记忆，{self.state.profile.facts[0]}。{style_hint}"
            return f"我还不知道你的名字，可以告诉我吗？{style_hint}"

        if "偏好" in user_input or "习惯" in user_input:
            if self.state.profile.preferences:
                return f"我记住了你的偏好：{self.state.profile.preferences}。{style_hint}"
            return f"我还没有记录你的偏好。{style_hint}"

        # 简单的任务意图
        if any(k in text for k in ["你好", "hi", "hello"]):
            return f"你好！我是你的 AI 学习助手。{style_hint}"
        if any(k in text for k in ["总结", "概括"]):
            return f"好的，我来总结一下。{style_hint}"

        # 默认回复
        return f"收到。我已经把这条信息加入上下文，后续会根据它来调整回答。{style_hint}"

    def reply(self, user_input: str) -> str:
        self.update_long_term_memory(user_input)

        if os.getenv("OPENAI_API_KEY"):
            assistant_reply = self._reply_with_llm(user_input)
        else:
            assistant_reply = self._reply_local(user_input)

        self.add_exchange(user_input, assistant_reply)
        self.save_state()
        return assistant_reply

    def reset_memory(self):
        """清空所有记忆（演示用户可控性）。"""
        self.state = AssistantState(user_id=self.user_id)
        self.save_state()


# ------------------- 演示 -------------------
if __name__ == "__main__":
    # 为了每次演示都是干净的，先删除旧状态文件
    if os.path.exists(STATE_FILE):
        os.remove(STATE_FILE)

    assistant = StatefulAssistant(user_id="skycc")

    demo_turns = [
        "你好，我是程峰，以后叫我峰哥就好。",
        "我喜欢简洁的回答，不要长篇大论。",
        "我在做 AI Native 产品的实战课程，今天是第四章。",
        "提醒我一下我是谁，以及我喜欢什么样的回答。",
        "今天我们学的主题是上下文即状态，你觉得它和 Prompt 即接口有什么关系？",
    ]

    for user_input in demo_turns:
        print("\n" + "=" * 60)
        print(f"🧑 用户：{user_input}")
        reply = assistant.reply(user_input)
        print(f"🤖 助手：{reply}")
        print(f"📦 当前状态摘要：{assistant.state.profile.summary or '（无）'}")

运行效果预览（无 API Key 版本）：

============================================================
🧑 用户：你好，我是程峰，以后叫我峰哥就好。
🤖 助手：你好！我是你的 AI 学习助手。
📦 当前状态摘要：已知事实：程峰

============================================================
🧑 用户：我喜欢简洁的回答，不要长篇大论。
🤖 助手：收到。我已经把这条信息加入上下文，后续会根据它来调整回答。（我会尽量简洁的回答）
📦 当前状态摘要：用户偏好：style=简洁的回答；已知事实：程峰

============================================================
🧑 用户：提醒我一下我是谁，以及我喜欢什么样的回答。
🤖 助手：根据我的记忆，程峰。（我会尽量简洁的回答）
📦 当前状态摘要：用户偏好：style=简洁的回答；已知事实：程峰

代码启示：StatefulAssistant 把「上下文」拆成了三层状态 —— 工作上下文（当前输入）、短期记忆（最近对话）、长期记忆（用户画像）—— 并把它们持久化到本地。无论底层用规则还是 LLM，调用方看到的都是同一个接口：reply(user_input)。

✅ 最佳实践

结合 2025–2026 年头部产品和开源社区的实践，总结出 7 条可操作的上下文状态设计指南：

  1. 把状态设计当作数据库 schema 来做 不要只用一个长字符串存「记忆」。定义清晰的结构：用户偏好、项目上下文、任务状态、环境变量。每个字段都要有类型、更新策略和过期策略。

  2. 区分短期记忆和长期记忆，不要混为一谈 短期记忆负责会话内的连贯性，可以塞满上下文；长期记忆负责跨会话的个性化，必须通过检索或摘要注入。混在一起会导致上下文膨胀和成本失控。

  3. 优先检索，而不是全量注入 即使是百万 token 上下文，也不要把整段历史都丢给模型。用向量检索、关键词匹配、时间衰减等策略，只召回与当前任务最相关的片段。

  4. 给用户记忆控制权 参考 Claude Memory 的做法：提供可查看、可编辑的记忆摘要；提供隐私/隐身模式；允许按项目/会话隔离记忆。用户不信任的记忆系统，用户不会持续使用。

  5. 设计状态边界，防止上下文污染 多租户、多项目、多会话之间必须隔离。不要把 A 项目的客户信息泄漏到 B 项目，也不要让一次实验性的对话永久污染用户画像。

  6. 利用缓存和压缩降低长上下文成本 生产环境中要开启 Prompt Caching；对重复使用的长上下文可以考虑 Hazy Research Cartridges 这类 KV Cache 压缩方案；对超长历史使用滚动摘要。

  7. 评估记忆是否真正帮助了任务 不要只测「模型记住了事实」，要测「记住这件事后，任务完成率、用户满意度、错误率有没有改善」。建立记忆质量的评估集，防止记忆变成噪声。
