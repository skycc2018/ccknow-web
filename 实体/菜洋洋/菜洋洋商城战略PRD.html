菜洋洋商城战略PRD：从B2B订货系统到AI Commerce OS

1. 文档概述

1.1 文档目的

本文档旨在明确菜洋洋商城的未来发展路线，基于铱云科技与 Shopify 的竞品分析，回答三个核心问题： 1. 菜洋洋到底该学谁？ 2. 先做什么？ 3. 3 年后怎么走到 AI Commerce OS？

1.2 核心结论（给老板的汇报版）

菜洋洋商城未来不应继续做"功能型商城系统"，而要分三步走：

第一步：做中国最强快消 B2B 订货商城（超越铱云） 第二步：做开放商业操作系统（对标 Shopify） 第三步：做 AI Commerce OS（领先国内同行）

一句话总结：

短期抄铱云补交易能力，中期抄 Shopify 建平台能力，长期超 Shopify 做 AI 原生商业操作系统。

这条路线，最符合菜洋洋现在的组织能力和技术积累。

2. 竞品定位：铱云 vs Shopify 本质区别

2.1 两种完全不同的商城哲学

维度 铱云科技
（国内产业互联网派）		 Shopify
（商业操作系统派）		 菜洋洋是什么样子的
核心定位 经销订货 + 渠道数字化 全渠道 Commerce OS 前期学铱云，中后期学 Shopify
服务对象 品牌商 / 经销体系 / B2B 批发 DTC / 品牌零售 / 全球商家 菜洋洋偏 B2B，应先铱云
架构思路 ERP 驱动商城 Commerce-first 平台 最终要 Shopify 化
商品模型 SKU 中心 Product Graph 中心 必须升级 Product Graph
营销能力 相对弱 极强（插件生态） 要补齐增长系统
SaaS 能力 多租户一般 真正云原生多租户 必须重构底座
开放生态 集成导向 Platform API First 必须做开放平台
AI 能力 辅助运营 Agentic Commerce 原生支持 必须提前布局

2.2 一句话定位

  • 铱云 = "渠道数字化软件"
  • Shopify = "商业基础设施"

菜洋洋的现实选择：** - 如果只学铱云 → 会成为传统订货系统 - 如果直接学 Shopify → 会因为基础能力不足而摔死

所以路线应该是：

第一阶段做铱云 → 第二阶段长成中国版 Shopify → 第三阶段进化 AI Commerce OS

3. 铱云软件竞品拆解（值得学的）

铱云厉害的不是页面，而是B2B 交易规则引擎和销售驱动模式。

3.1 订货交易闭环（第一优先级）

核心能力： - 客户分层 - 客户价盘 - 渠道价体系 - 授信 - 账期 - 配送规则 - 返利 - 对账

本质： B2B 交易规则引擎

菜洋洋目前问题： 如果商城还是"普通下单页"，会很弱。

必须建设：交易规则中心

交易规则中心
├── 客户等级
├── 客户协议价
├── 区域限制
├── SKU 可售范围
├── 促销规则
├── 信控规则
├── 返利策略
└── 履约优先级

3.2 销售驱动商城

铱云强在： 销售员不是旁观者，而是参与者： - 代客下单 - 客户推荐 - 跟单 - 客户行为洞察

与 CRM 规划高度契合

菜洋洋必须做：Sales-assisted Commerce

AI 销售助手能力
├── AI 推荐今日该联系谁
├── AI 生成沟通建议
├── 客户下单预测
├── 缺货提醒
├── 商机预警
└── 客户画像自动构建

这会形成护城河。

3.3 渠道运营中台

铱云： 经销商数字化管理很强

菜洋洋可以升级成：智能渠道运营中台

智能渠道运营中台
├── 区域渗透分析
├── 品类空白识别
├── 自动招商建议
├── 销售路径优化
├── 经销商健康度评分
└── 渠道冲突预警

这个会比铱云先进。

4. Shopify 拆解（真正该抄的）

Shopify 2026 已经不是建站工具，而是 Commerce OS。

核心是 5 层架构：

4.1 Layer 1：统一商品底座（Product Graph）

Shopify 的商品不是 SKU 表，而是：

Product Graph
├── 商品
│   ├── 属性
│   ├── 变体
│   ├── 价格
│   ├── 库存
│   ├── 内容
│   ├── 媒体
│   ├── SEO
│   ├── 渠道映射
│   └── 推荐关系

菜洋洋必须重构：

# 从
sku_basic

# 升级
product_graph
├── product_attribute
├── product_relation
├── product_embedding
└── channel_mapping

这是 AI 选品基础。

4.2 Layer 2：应用市场（App Marketplace）

Shopify 最大壁垒：生态

商家缺功能 → 直接装 App

菜洋洋未来必须支持：

App Marketplace
├── BI 插件
├── CRM 插件
├── AI 插件
├── 营销插件
├── 仓储插件
└── 渠道插件

底层支撑：
├── OAuth
├── Webhook
├── Event Bus
└── SDK

4.3 Layer 3：Flow Engine（自动化）

Shopify Flow 很强。

菜洋洋要做：规则自动化引擎

when:
  客户 30 天未下单

then:
  生成销售任务
  推送优惠策略
  推荐替代品
  提醒销售跟进

这是效率爆发点。

4.4 Layer 4：Sidekick（AI Copilot）

Shopify Sidekick 已经开始进入执行型 AI。

菜洋洋未来必须有：菜小羊 Agent

示例：

用户说："最近华东饮料怎么补？"

AI 自动： 1. 分析销量 2. 判断库存 3. 推荐采购 4. 给销售生成策略 5. 一键执行

这就是 AI OS 雏形。

4.5 Layer 5：Agentic Commerce

未来流量入口不是商城首页，而是 AI 对话入口。

示例：

客户直接问："给我推荐利润最高且周转快的矿泉水"

AI 完成交易。

菜洋洋必须提前准备： - 商品语义索引 - 商品知识库 - MCP 接口 - Agent API

6. 给老板的最终汇报结论

6.1 核心观点

菜洋洋商城未来不应继续做：

"功能型商城系统"

而要分三步：

第一步：做中国最强快消 B2B 订货商城（超越铱云）
        ↓
第二步：做开放商业操作系统（对标 Shopify）
        ↓
第三步：做 AI Commerce OS（领先国内同行）

6.2 一句话总结

短期抄铱云补交易能力，中期抄 Shopify 建平台能力，长期超 Shopify 做 AI 原生商业操作系统。

6.3 路线合理性

这条路线，最符合菜洋洋现在的： - 组织能力 - 技术积累 - 客户基础 - 行业认知
