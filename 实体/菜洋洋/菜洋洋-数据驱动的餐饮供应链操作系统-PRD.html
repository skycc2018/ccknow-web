菜洋洋 — 数据驱动的餐饮供应链操作系统 PRD

版本：V1.0
日期：2026-06-22
策略来源：对标窄门餐眼，从"数据工具"升级为"交易操作系统"
核心定位：爆品 → 一键补货

1. 项目背景与机会

1.1 行业现状

窄门餐眼是中国餐饮行业的典型数据中台，主要解决看的问题： - 品牌排名、门店分布、开关店趋势 - 外卖销量、评价分析、热销菜品 - 品类爆款、城市机会、竞品动态

但窄门的瓶颈在于： - 数据仅停留在"信息展示"层 - 用户看完数据后，仍需要离开平台去开展后续行动 - 未掌握订单流、资金流、SKU 流

1.2 菜洋洋的机会

菜洋洋的优势在于： - 已有供应链基础（食材采购、物流、仓储） - 服务大量餐饮门店客户 - 具备"从数据到交易"的产业链位置

机会点：

从"让你知道该做什么"，升级为"帮你直接做完"

2. 产品定位

2.1 一句话定位

菜洋洋数据驱动的餐饮供应链操作系统

2.2 与窄门的差异

维度 窄门餐眼 菜洋洋（本系统）
数据源 公开数据拼接 门店真实经营数据 + 公开数据
价值 降低认知成本 降低决策和执行成本
模式 SaaS 数据工具 交易操作系统
核心闭环 数据 → 报告 数据 → 决策 → 交易 → 配送 → 复盘
收入 会员费 供应链抽成 / SaaS 订阅 / 金融服务
天花板 信息服务 产业基础设施

2.3 产品特色

  1. 门店级数据：POS 、收银系统、进销存、毛利结构
  2. 决策引擎：AI 告诉门店"该上什么菜、该补什么货、该换什么供应商"
  3. 一键执行：从数据决策直接下单到供应链
  4. 交易闭环：选品 → 比价 → 下单 → 配送 → 复购 → 金融

3. 目标用户

3.1 核心用户

用户角色 痛点 价值
餐饮门店老板 不知道该上什么菜、该进多少货 数据驱动的采购建议
门店采购员 每天花大量时间比价、下单 一键补货、自动比价
连锁品牌总部 无法实时掌握各门店营销和库存 多门店统一视图和调配
供应商 缺少稳定订单渠道 获得准确的采购需求
菜洋洋运营团队 需要提升客户粘性和增值 从供应商升级为产业平台

3.2 用户分层

  • 小型单店：重点需求是"爆品发现 + 一键补货"
  • 中型连锁（10-50 家）：重点需求是"多店统一采购 + 品类管理"
  • 大型连锁（50+ 家）：重点需求是"数据中台 + AI 决策 + 供应链金融"

4. 核心价值主张

4.1 主张一：爆品 → 一键补货

核心逻辑：通过门店销量数据和外部趋势数据，自动识别爆品，并推荐应该补货的 SKU 和数量。

功能路径： 1. 接入门店 POS/收银系统 2. 自动识别门店爆品（日、周、月维度） 3. 结合外部趋势（同城同类热销菜品） 4. 推荐补货 SKU 、数量、供应商 5. 用户一键确认，自动生成采购单 6. 同步到菜洋洋供应链执行

4.2 主张二：智能采购建议

基于销量、库存、天气、节假日、供应商价格等因素，预测采购需求。

关键场景： - "这周咨询量下降，是不是该换一个 SKU？" - "明天下雨，点单量可能上涨，提前备多少货？" - "这个供应商价格涨了 8%，要不要换一家？"

4.3 主张三：供应链比价与优化

集合多个供应商报价，根据价格、品质、交付周期、历史合规率自动推荐最优采购方案。

4.4 主张四：多门店统一视图

连锁品牌可以统一查看各门店的销量、库存、毛利、缺货情况，并统一调配采购。

5. MVP 范围（第一阶段）

5.1 MVP 核心目标

验证：门店愿意为"数据驱动的一键补货"付费

5.2 MVP 功能清单

模块 功能 说明
数据接入 POS 接入 至少支持 1-2 家主流 POS 厂商（如客如云、美团收银等）
进销存手工录入 作为 POS 数据的补充
外部趋势数据 同城热销菜品、竞品菜单
爆品识别 门店爆品排行 按日/周/月统计 SKU 销量 TOP
趋势爆品 同比/环比增长超过阈值的 SKU
外部爆品 同城同类热销菜品
补货推荐 推荐采购单 根据销量、库存、安全库存推荐数量
多供应商比价 自动对比菜洋洋平台内供应商报价
一键下单 确认后自动生成订单并同步供应链
数据看板 门店经营概览 销量、客单价、毛利率、缺货率
采购分析 采购金额、供应商占比、价格趋势
基础设施 用户体系 门店、采购员、角色权限
供应商接入 供应商入驻、商品上架、报价

5.3 明确不做的（MVP 阶段）

  • 外卖平台数据抓取（第一阶段不做，只做 POS 接入）
  • AI 智能决策（先做规则化推荐）
  • 供应链金融
  • 多城市、多仓复杂配送
  • 私域社区/评论分析

6. 功能模块详细设计

6.1 数据接入层

6.1.1 POS 数据接入

  • 支持常见 POS 厂商 API 接入
  • 定期同步订单明细（商品、数量、金额、时间）
  • 缺少 API 时支持导入 Excel

6.1.2 进销存数据

  • 日常进货录入
  • 盘点数据录入
  • 实时库存查询

6.1.3 外部趋势数据（第二阶段）

  • 同城热销菜品
  • 竞品菜单变化
  • 品类增长趋势

6.2 爆品识别引擎

6.2.1 算法规则

爆品类型 判断规则
稳定爆品 近7天销量 TOP10 且连续 3 天在前 10
趋势爆品 同比增长 > 30% 或 环比增长 > 50%
潜力爆品 新上架 SKU 中 7 天内销量进前 20
外部爆品 同城同类门店热销 TOP5

6.2.2 输出结果

  • 爆品名单
  • 爆品指数（根据销量、增速、毛利综合计算）
  • 爆品原因解释

6.3 补货推荐引擎

6.3.1 推荐逻辑

建议采购量 = max(
    预测未来5天销量 - 当前库存,
    安全库存 - 当前库存,
    0
)

6.3.2 供应商推荐

  • 优先推荐历史合作过的供应商
  • 同品类价格最优
  • 交付周期符合要求
  • 评分高的供应商

6.4 一键补货流程

  1. 系统生成采购建议单
  2. 采购员确认/修改
  3. 系统自动拆分订单给供应商
  4. 供应商确认并发货
  5. 仓库/门店验收
  6. 自动更新库存

6.5 数据看板

6.5.1 门店经营看板

  • 今日销量、客单价、毛利率
  • 热销 SKU TOP10
  • 缺货 SKU 预警
  • 库存周转天数

6.5.2 采购看板

  • 本月采购金额
  • 供应商占比
  • 价格波动趋势
  • 采购频次分析

6.6 供应商端

  • 商品上架与报价
  • 订单接收与发货
  • 账款查询
  • 销量统计

7. 数据架构与指标体系

7.1 核心数据模型

实体 主要字段
门店 门店 ID、名称、位置、经营品类、POS 类型
SKU SKU ID、名称、品类、规格、单位、成本价
订单明细 时间、门店、SKU、数量、金额
库存 SKU、当前库存、安全库存、在途数量
供应商 供应商 ID、名称、联系人、评分
供应商报价 SKU、价格、起订量、交付周期
采购建议单 门店、SKU、建议量、推荐供应商、状态
采购订单 供应商、SKU、数量、价格、状态

7.2 核心指标体系

7.2.1 销售类

  • 日/周/月销量
  • 客单价（AOV）
  • 同比增长率
  • 环比增长率
  • SKU 销量排行
  • 品类销量占比

7.2.2 库存类

  • 库存周转天数
  • 缺货率
  • 安全库存符合率
  • 残次品占比

7.2.3 采购类

  • 采购金额
  • 采购频次
  • 供应商集中度
  • 价格波动率
  • 一键补货占比

7.2.4 爆品类

  • 爆品指数
  • 爆品识别准确率
  • 爆品与采购转化率

8. 非功能性需求

8.1 技术架构

  • 前端：Vue 3 / React + TypeScript
  • 后端：Spring Boot / Go 微服务
  • 数据库：PostgreSQL + Redis
  • 数据仓库：ClickHouse / Doris（适合分析型查询）
  • 消息队列：RabbitMQ / Kafka
  • 数据接入：ETL 工具 + API 网关

8.2 性能

  • 数据看板加载 < 3s
  • 爆品计算定时任务 T+1 出结果
  • 支持 1000+ 门店数据量

8.3 安全

  • 门店数据隔离
  • 金融级数据加密
  • POS 接入 OAuth2 认证
  • 操作日志审计

9. 商业模式

9.1 收入来源

模式 说明
SaaS 订阅 门店按月/年付费使用数据驱动采购系统
供应链抽成 通过平台产生的交易金额抽成
金融服务 采购贷、供应链信贷（第三阶段）
数据服务 行业报告、品类分析（第三阶段）

9.2 客户价值

  • 降低缺货率 10%-20%
  • 减少采购人员日常工作时间 30%+
  • 提升库存周转效率
  • 通过数据优化毛利结构

10. 里程碑

阶段 周期 主要交付物
M1 需求与架构 PRD 确认、技术选型、数据模型、POS 接入方案
M2 MVP 开发 数据接入、爆品识别、补货推荐、一键下单
M3 内测与优化 5-10 家门店试点、收集反馈、算法调优
M4 扩展功能 多门店、外部趋势数据、AI 决策引擎
M5 商业化 定价体系、供应商拓展、运营体系
M6 扩大覆盖 多城市推广、持续迭代

11. 成功标准

11.1 过程指标

  • 100+ 门店接入 POS 数据
  • 80% 以上门店每周至少使用 1 次一键补货
  • 缺货率下降 10%+

11.2 结果指标

  • 平台月交易金额达到 XXX 万
  • SaaS 付费门店数达到 XXX 家
  • 供应商入驻达到 XXX 家
