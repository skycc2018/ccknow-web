1.2 核心设计原则

原则 说明
中台唯一 供应链中台必须收敛为一套，商品/库存/价格/履约逻辑不 fork
前台多样 自营商城、SaaS 商城、租户后台都是中台上的不同前台应用
租户隔离 买方侧数据按 tenant_id 隔离，平台级资源共享
供应商开放 支持多供应商接入，菜洋洋是默认深度集成供应商
渐进收敛 设计上为自营收敛预留，实际迁移待 SaaS 验证稳定后执行

2.2 关键页面详细设计

2.2.1 开户向导（核心流程）

页面流程：

Step 1: 创建租户
  ├── 租户名称
  ├── 企业信息（营业执照/联系人）
  ├── 分配套餐（门店数/站点数/供应商数/选品数）
  └── 有效期设置

Step 2: 绑定站点
  ├── 选择可用站点（多选）
  ├── 设置默认站点
  └── 校验站点绑定数上限

Step 3: 订阅供应商
  ├── 自动订阅公共供应商（菜洋洋）
  ├── 自动创建租户私有供应商
  └── 可选订阅其他公共供应商

Step 4: 平台代选品
  ├── 从平台商品库批量选品
  ├── 按类目/品牌筛选
  └── 生成租户选品记录

Step 5: 直设租户站点价（L4）
  ├── 批量设置站点价
  ├── 应用默认价格模板
  └── 生成L4价格快照

Step 6: 配置默认价格模板
  ├── 加价策略类型（固定加价/百分比/公式）
  ├── 策略参数
  └── 生效范围

Step 7: 创建租户管理员账号
  ├── 管理员姓名/手机
  ├── 初始密码
  └── 发送通知

业务规则： - 开户完成后租户状态 = 正常(1)，可立即采购 - 平台代选品经 @CrossTenantAccess 标记，留审计日志 - 私有供应商自动创建，supplier_type = 'PRIVATE', tenant_id = 当前租户

3.2.1 自建商品管理

创建流程：

Step 1: 基础信息
  ├── 商品名称
  ├── 商品编码（自动生成/自定义）
  ├── 类目（复用平台类目或自定义）
  ├── 品牌
  ├── 单位/规格/重量
  └── 图片

Step 2: 履约方式选择
  ├── 平台配送（仓配代发）
  │   └── 需要平台ERP建档 → 提交审核
  └── 自配送
      └── 免审，直接上架

Step 3: 价格设置
  ├── L1成本价（可选填）
  ├── L4租户站点价（必填）
  └── L5门店价（必填/跟随站点价）

Step 4: 库存设置
  ├── 初始库存（自配送需手工维护）
  └── 库存预警阈值

Step 5: 站点分配
  ├── 选择已绑定站点
  ├── 设置站点价
  └── 上架状态

业务规则： - 自建商品挂本租户私有供应商下（supplier_id = 私有供应商ID） - 仅本租户可见，他租户任何接口不可见 - 仓配代发商品建档启用前不可上架（status = 待建档） - 创建后自动生成选品记录

3.2.2 加价策略管理

策略优先级：

SKU级策略 > 站点级策略 > 租户默认模板

策略类型：

类型 说明 示例
固定加价 在基准价上加固定金额 L2 + 5元
百分比加价 在基准价上加百分比 L2 * 1.15
公式定价 自定义公式 max(L2 * 1.1, L2 + 3)
指定价 直接指定价格 不跟随基准

计算流程：

输入: SKU + Site + Tenant
  ↓
查找 SKU级策略?
  → 有: 应用SKU策略 → 输出 L4
  → 无: 继续
  ↓
查找 站点级策略?
  → 有: 应用站点策略 → 输出 L4
  → 无: 继续
  ↓
应用 租户默认模板 → 输出 L4
  ↓
门店价 L5 = L4 + 门店微调（可选）

4.2 核心流程设计

4.2.1 采购下单流程

[采购员] 登录 → 搜索商品 → 加入采购车 → 提交订单
  ↓
[系统] 校验：
  - 门店归属 tenant_id
  - 站点绑定 site_id
  - 供应商订阅
  - 选品状态
  - 商品上下架
  - 门店价有效性
  - 站点库存充足
  ↓
[系统] 锁定库存（available - qty, locked + qty）
  ↓
[系统] 命中审批规则?
  → 是: 创建审批实例，订单展示"待审批"
  → 否: 进入待付款
  ↓
[审批人] 审批通过 → 解锁支付
  ↓
[采购员] 发起微信支付（wx.b2b）
  ↓
[系统] 支付回调 → 标记已支付
  - 扣减锁定库存（locked - qty）
  - 发送 OrderPaidEvent
  - 推单到 ERP（fulfillment_type=1）

4.2.2 混合履约拆单流程

采购车商品：
├── 商品A（平台商品）→ 平台配送
├── 商品B（已建档自建商品）→ 平台配送
└── 商品C（自配送自建商品）→ 自配送

提交订单 → 按履约方式拆单：
├── 子单1（root_id=X, fulfillment_type=1）
│   ├── 商品A + 商品B
│   └── 推ERP
└── 子单2（root_id=X, fulfillment_type=2）
    ├── 商品C
    └── 租户自配送

合并收银：
├── 支付单金额 = 子单1金额 + 子单2金额
├── 支付回调 → 逐子单标记已支付
└── 仅子单1推ERP

4.3 数据范围约束

约束项 规则
商品可见 仅本租户 + 本门店所属站点 + 已上架
价格展示 仅 L5 门店价，不暴露 L1-L4
订单范围 仅本门店采购的订单
售后范围 仅本门店发起的售后

-- 站点可售库存（下单锁定）
CREATE TABLE t_site_stock (
    id BIGINT PRIMARY KEY,
    tenant_id BIGINT NOT NULL,
    site_id BIGINT NOT NULL,
    sku_id BIGINT NOT NULL,
    available_stock INT DEFAULT 0, -- 可售库存
    locked_stock INT DEFAULT 0,    -- 锁定库存
    warning_threshold INT DEFAULT 0, -- 预警阈值
    update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE KEY uk_tenant_site_sku (tenant_id, site_id, sku_id)
);

库存操作流程：

1. 下单/提交审批: 
   available_stock -= qty
   locked_stock += qty

2. 审批驳回/取消/支付超时:
   available_stock += qty
   locked_stock -= qty

3. 支付成功:
   locked_stock -= qty (转已售/出库)
   -- available_stock 不变（已在步骤1扣减）

4. ERP SyncStock回写:
   更新供应商库存账本 → 触发站点库存对账/刷新

6.2 多供应商策略

核心决策：支持多供应商接入，菜洋洋作为默认深度集成供应商。

供应商生态：
├── 平台公共供应商
│   ├── 菜洋洋（默认，深度集成）
│   ├── 第三方供应商A（未来接入）
│   └── 第三方供应商B（未来接入）
└── 租户私有供应商
    ├── 租户1私有（自建商品载体）
    └── 租户2私有（自建商品载体）

接入策略：

维度 菜洋洋 第三方供应商
接入方式 原生深度集成 Provider SPI 扩展
商品同步 ERP双向同步 按需接入
库存同步 实时同步 按需接入
订单推单 自动路由 配置路由
售后同步 完整闭环 按需接入
差异化 出品包/独家SKU 通用商品

竞争优势设计： - 菜洋洋不靠"排他锁定"占份额，靠出品包和差异化 SKU - 优质连锁可接入自有中央厨房、自有供应商 - 多供应商同 SKU 比价（v3）

8. 权限与安全设计

8.1 三层 Token 体系

Token类型 使用方 携带信息 隔离级别
Platform JWT 平台运营 用户ID + 角色 无租户隔离
Tenant JWT 租户管理员/运营 用户ID + tenantId + 角色 租户级隔离
Store JWT 门店采购员 用户ID + tenantId + shopId + shopAccountId 门店级隔离

8.2 数据范围控制

// 租户级隔离示例
@TenantIsolation
public List<Order> queryOrders(OrderQueryParam param) {
    // 自动注入当前租户ID
    Long tenantId = TenantContext.getCurrentTenantId();
    param.setTenantId(tenantId);
    return orderMapper.selectList(param);
}

// 平台跨租户访问（需审计）
@CrossTenantAccess(reason = "开户向导代选品")
public void batchSelectProducts(Long targetTenantId, List<Long> skuIds) {
    // 记录审计日志
    auditLog.record("BATCH_SELECT", targetTenantId, skuIds);
    // 执行跨租户写操作
    selectionService.batchSelect(targetTenantId, skuIds);
}

8.3 敏感字段控制

敏感字段 可见角色 控制方式
L1成本价 仅平台运营 VO层过滤，接口不返回
ERP凭据 仅平台运营 加密存储，接口不返回
Provider协议字段 仅mall-erp 业务服务不可见
租户交易数据 仅本租户 租户ID强制过滤

8.4 安全红线

  1. L1成本价不得经买方端VO/接口返回给门店
  2. 跨租户数据读取必须经 @CrossTenantAccess + 审计日志
  3. ES查询必须强制 tenant_id + site_id 过滤
  4. 定时任务和MQ消费必须手动设置并清理租户上下文
  5. 支付回调、ERP入站、AI执行必须有审计日志
