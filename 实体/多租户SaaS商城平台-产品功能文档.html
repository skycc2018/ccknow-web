多租户SaaS商城平台 - 产品功能文档

1. 产品概述与定位

1.1 产品定位

产品名称：UniShop SaaS 商城平台 产品定位：面向中小微企业和个体商户的一站式多租户电商SaaS解决方案 核心价值：让商户在5分钟内快速搭建独立品牌商城，无需技术投入即可拥有专业级电商能力

1.2 目标用户

  • 中小微电商企业：年GMV 50万-5000万，无技术团队
  • 品牌零售商：希望快速拓展线上渠道
  • 个体创业者：微商、代购、内容创作者变现
  • 连锁门店：总部统一管理，门店独立运营

1.3 产品愿景

成为中小微商户最信赖的电商基础设施，提供"开店即经营"的极致体验。

2. 多租户架构设计

2.1 架构模式：共享数据库 + 租户ID隔离（推荐）

┌─────────────────────────────────────────┐
│           接入层 (Gateway)               │
│  - 域名路由: shop1.unishop.com          │
│  - 租户识别: Header / Subdomain / Path  │
│  - 流量控制: 按租户限流                  │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│           应用层 (Service)              │
│  - 租户上下文传递: TenantContext         │
│  - 数据自动过滤: WHERE tenant_id = ?   │
│  - 缓存隔离: tenant_id:key              │
└──────────────┬──────────────────────────┘
               │
┌──────────────▼──────────────────────────┐
│           数据层 (Database)             │
│  - 共享数据库 + 租户ID字段隔离            │
│  - 索引优化: (tenant_id, business_key)  │
│  - 分表策略: 大数据量时按租户分表         │
└─────────────────────────────────────────┘

2.2 租户隔离策略

隔离维度 实现方式 说明
数据隔离 行级隔离（tenant_id字段） 所有业务表含tenant_id，查询自动注入条件
配置隔离 独立配置表 + 租户覆盖 系统默认配置 + 租户自定义覆盖
文件隔离 对象存储路径隔离 /{tenant_id}/uploads/...
缓存隔离 Key前缀隔离 cache:{tenant_id}:{key}
队列隔离 队列名称/Tag隔离 租户级别消息分组

2.3 租户生命周期管理

注册 → 试用(14天) → 付费订阅 → 正常运营 → 续费/升级/降级 → 到期提醒 → 暂停/恢复 → 注销

状态定义： - trial：试用期，功能全开，带水印 - active：正常付费运营 - suspended：欠费暂停，仅保留数据，前端展示维护页 - expired：到期未续费，进入30天保留期 - terminated：彻底注销，数据归档（保留90天后删除）

3. 核心功能模块

3.1 租户管理后台（Tenant Admin）

3.1.1 店铺配置

  • 基础信息：店铺名称、Logo、简介、客服电话、ICP备案
  • 域名管理：
  • 默认子域名：{shop_id}.unishop.com
  • 自定义域名绑定：支持CNAME，自动SSL证书（Let's Encrypt）
  • 域名备案状态检测
  • 地区/语言：多语言支持（中/英/日/韩），多币种（CNY/USD/EUR）
  • 营业时间：可设置配送/自提营业时间
  • 物流配置：默认运费模板、发货地址管理

3.1.2 主题/装修管理

  • 主题市场：官方主题库（免费+付费），支持一键切换
  • 可视化装修：
  • 拖拽式页面编辑器（类似Shopify Sections）
  • 组件库：轮播图、商品列表、图文、视频、倒计时、优惠券领取
  • 页面管理：首页、分类页、商品详情、自定义页
  • 自定义CSS/JS：高级用户可注入自定义样式（需安全审核）
  • 预览与发布：草稿/发布双版本，支持多端预览

3.1.3 支付与财务

  • 支付渠道配置：
  • 微信支付（需商户自行申请）
  • 支付宝
  • 银联云闪付
  • 国际：Stripe、PayPal
  • 分账设置：平台抽成比例、自动分账规则
  • 结算周期：T+7/T+14/T+30，自动结算到绑定银行卡
  • 发票管理：电子发票申请、开票记录

3.1.4 员工与权限

  • 角色系统：超级管理员、运营、客服、财务、仓管
  • 权限矩阵：菜单权限 + 操作权限 + 数据权限（仅看自己/看全部）
  • 操作日志：关键操作留痕，支持导出审计

3.2 商品管理系统（PMS）

3.2.1 商品模型

SPU（标准产品单元）
├── 基础信息：名称、副标题、品牌、分类、主图、视频、详情
├── 销售属性：颜色、尺码、规格 → 组合生成SKU
├── 扩展属性：材质、产地、保质期等（按类目模板）
├── 图文详情：富文本编辑器，支持组件化模板
└── 营销标签：新品、热销、推荐、限时

SKU（库存量单元）
├── 规格组合：{颜色:红, 尺码:XL}
├── 价格体系：售价、原价、成本价、会员价
├── 库存：独立库存/共享库存，支持多仓
├── 条码：SKU编码、条形码、二维码
└── 图片：SKU专属图（如不同颜色展示不同图）

3.2.2 分类与品牌

  • 类目体系：
  • 平台标准类目（3级）：后台维护，租户选用
  • 租户自定义类目：可映射到平台类目，支持独立展示
  • 品牌管理：品牌库（平台预置 + 租户自建），品牌授权资质上传
  • 类目属性模板：不同类目自动加载对应规格属性（如服装有尺码表，食品有保质期）

3.2.3 库存管理

  • 库存类型：实物库存、虚拟库存、预售库存
  • 库存操作：入库、出库、盘点、调拨、报损
  • 库存预警：低库存提醒、缺货自动下架
  • 多仓支持：
  • 仓库管理：主仓、分仓、门店仓
  • 库存分配规则：就近发货、库存均衡、指定仓库
  • 库存同步：实时同步/定时同步

3.2.4 商品状态流转

草稿 → 待审核（可选） → 上架 → 在售 → 售罄 → 下架 → 删除

3.2.5 批量操作

  • Excel导入/导出（标准化模板）
  • 批量修改：价格、库存、分类、上下架
  • 批量图片上传（压缩、水印、CDN加速）

3.3 订单管理系统（OMS）

3.3.1 订单状态机

待付款 → 已付款 → 待发货 → 已发货 → 待收货 → 已完成
   ↓        ↓         ↓         ↓          ↓
 已取消   退款中    退款完成   退货中     退货完成

3.3.2 核心功能

  • 订单列表：多维度筛选（状态、时间、来源、金额），高级搜索
  • 订单详情：完整信息展示、操作日志、物流跟踪
  • 发货管理：
  • 电子面单：对接菜鸟、快递100，自动打印
  • 批量发货：Excel导入运单号
  • 拆单/合单：多商品拆开发货、多订单合并发货
  • 无需物流：虚拟商品自动发货
  • 物流跟踪：对接主流快递，实时轨迹查询，异常预警

3.3.3 售后管理

  • 退款/退货流程：
  • 仅退款：未发货时买家申请
  • 退货退款：已发货后申请，填写物流单号
  • 换货：库存充足时支持
  • 售后审核：商家审核 → 买家退货 → 商家确认收货 → 退款
  • 退款路径：原路退回（微信/支付宝/银行卡）
  • 纠纷处理：平台介入仲裁（针对平台自营模式）

3.3.4 订单来源与标记

  • 来源渠道：H5、小程序、App、PC、线下POS
  • 订单标记：加急、赠品、大客户、备注标签
  • 自动规则：自动打标签（如首单、高客单、复购）

3.4 会员/用户系统（CRM）

3.4.1 用户体系

  • 用户注册：手机号+验证码、微信授权、Apple ID（iOS）
  • 用户画像：基础信息、收货地址、浏览偏好、消费特征
  • 用户分层：
  • 按消费：普通会员、银卡、金卡、钻石
  • 按RFM：重要价值、重要保持、新客户、流失预警
  • 按行为：活跃、沉默、沉睡、流失

3.4.2 会员等级

  • 成长值体系：消费1元=1成长值，签到、评价额外获得
  • 等级权益：
  • 折扣权益：不同等级享受不同折扣
  • 服务权益：优先发货、专属客服、生日礼
  • 营销权益：专属优惠券、提前购、专享价
  • 等级有效期：按自然年/滚动年计算，到期降级

3.4.3 积分系统

  • 积分获取：消费返积分、签到、分享、评价、完善资料
  • 积分消耗：抵扣现金（100积分=1元）、兑换商品、抽奖
  • 积分规则：商户自定义比例、有效期、使用门槛
  • 积分明细：完整流水记录

3.4.4 用户标签

  • 自动标签：基于行为自动打标（高客单、爱折扣、夜间购物）
  • 手动标签：运营人工标注（大客户、KOC、投诉用户）
  • 标签应用：精准营销、分组推送、个性化展示

3.4.5 用户运营工具

  • 群发消息：按标签/分组推送短信/站内信/订阅消息
  • 关怀提醒：生日祝福、沉默唤醒、购物车召回
  • 黑名单：恶意用户、羊毛党标记

3.5 营销系统

3.5.1 优惠券

  • 券类型：
  • 满减券：满X元减Y元
  • 折扣券：X折，最高减Y元
  • 兑换券：凭码兑换指定商品
  • 运费券：免邮/减邮费
  • 发放方式：
  • 主动领取：领券中心、商品详情页
  • 被动发放：新客注册、生日、消费返券、补偿
  • 定向发放：指定用户/标签群发
  • 使用规则：
  • 适用范围：全店/指定商品/指定分类
  • 叠加规则：是否可叠加、互斥组
  • 有效期：固定日期/领取后N天
  • 使用门槛：最低消费金额、指定会员等级

3.5.2 满减/满折

  • 阶梯满减：满100减10、满200减30、满300减50
  • 每满减：每满100减10，上不封顶
  • 满件折：满2件8折、满3件7折
  • 满赠：满额赠商品/赠品池随机

3.5.3 限时秒杀

  • 场次管理：固定场次（10:00/14:00/20:00）或自定义
  • 商品设置：秒杀价、库存、限购数量、预热时间
  • 防超卖：独立秒杀库存，与主库存隔离
  • 高并发：库存扣减优化（Redis预扣 + 异步落库）

3.5.4 拼团

  • 团类型：普通团（2-5人）、阶梯团（人越多越便宜）、新人团
  • 成团规则：成团时间、人数、虚拟成团（差1人自动成团）
  • 分享机制：海报分享、小程序卡片、链接

3.5.5 分销/推广员

  • 分销模式：
  • 一级分销：A推广B，A得佣金
  • 二级分销：A推广B，B推广C，A和B都得佣金（合规风险需注意）
  • 推广员管理：申请审核、等级体系、佣金比例
  • 佣金结算：订单完成后结算，可提现到微信零钱
  • 推广工具：专属海报、推广链接、二维码

3.5.6 直播带货（可选高级功能）

  • 直播间管理、商品上架、优惠券发放、弹幕互动
  • 回放功能、直播数据统计

3.5.7 营销工具箱

  • 签到有礼：连续签到奖励递增
  • 抽奖/转盘：积分抽奖、消费抽奖

6. 安全与合规

6.1 数据安全

  • 传输加密：全站HTTPS（TLS 1.3），HSTS头部
  • 存储加密：
  • 敏感字段加密（手机号、身份证、银行卡）
  • 数据库TDE透明加密（可选）
  • 配置文件加密（Nacos配置中心加密）
  • 密码安全：BCrypt加密，禁止明文存储
  • 密钥管理：阿里云KMS / AWS KMS，定期轮换

6.2 接口安全

  • 认证方式：JWT Token + Refresh Token
  • 防重放攻击：请求时间戳 + 随机数校验
  • 防篡改：请求签名（HMAC-SHA256）
  • 限流熔断：
  • 网关层：IP限流（100次/分钟）
  • 租户级：按套餐配额限流
  • 接口级：敏感接口额外限流（如支付、发短信）

6.3 业务安全

  • 防刷单：设备指纹、行为分析、异常订单拦截
  • 防羊毛党：新人标识、同设备/同IP限制、黑名单
  • 库存安全：Redis原子扣减 + 数据库乐观锁，防止超卖
  • 支付安全：
  • 支付密码/指纹/面容验证
  • 大额支付二次确认
  • 异常支付风控拦截

6.4 合规要求

  • 等保三级：符合中国网络安全等级保护要求
  • 数据隐私：
  • 用户协议与隐私政策
  • 数据最小化原则
  • 用户数据导出与删除（GDPR/个人信息保护法）
  • 内容审核：商品信息敏感词过滤、图片鉴黄（阿里云内容安全）
  • 电商合规：
  • 7天无理由退货（法定）
  • 发票合规（电子发票）
  • 价格合规（禁止价格欺诈）

  • ARPU：Average Revenue Per User，每用户平均收入
  • RFM：Recency, Frequency, Monetary，用户价值分析模型
  • RBAC：Role-Based Access Control，基于角色的访问控制

B. 接口规范

  • RESTful API设计规范
  • 统一响应格式：{code, message, data, timestamp}
  • 分页规范：{page, pageSize, total, list}
  • 错误码规范：1xxxx系统级、2xxxx业务级、3xxxx第三方
