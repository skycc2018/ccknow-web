一、你的角色

你是一名资深全栈产品开发工程师，需要根据以下产品需求，生成一个可运行的Web端MVP原型。

产品名称：AI商机雷达
英文名：Opportunity AI
产品定位：面向餐饮供应链B2B销售的AI客户发现系统。
核心目标：帮助销售输入自己销售的产品后，系统自动推荐最值得优先开发的餐饮客户，并展示商机评分、推荐理由、客户画像和销售话术。

二、技术栈要求

请使用以下技术栈开发：

前端：
- Next.js
- TypeScript
- Tailwind CSS
- Shadcn UI
- Lucide Icons

数据：
- 本地Mock数据
- 暂不接真实后端

状态管理：
- React Hooks
- 可使用本地状态管理

图表：
- 使用简洁卡片和基础图表即可
- 不需要复杂可视化

本阶段目标是生成一个可运行、可演示、可交互的前端MVP原型，不是完整生产系统。

三、产品核心逻辑

用户输入自己销售的产品，例如：

牛肉卷
芝士碎
鸡翅中
调味料
冻品

系统根据产品匹配适合开发的餐饮客户类型，并推荐客户列表。

示例：

用户输入：牛肉卷

系统推荐客户类型：
- 火锅
- 烤肉
- 自助餐
- 韩餐

系统输出：
- 上海火锅TOP商机客户
- 杭州烤肉TOP商机客户
- 全国高潜客户排行榜

推荐理由

2. 增长信号：近期门店增长明显，处于扩张阶段。
3. 品类匹配：主营火锅场景，与牛肉卷、冻品类产品匹配度较高。
4. 切入建议：建议从稳定供货、标准化食材和成本优化角度切入。

七、商机评分模型

请在前端Mock中实现评分展示逻辑。

综合商机指数公式：

商机指数
= 增长潜力 × 25%
+ 规模价值 × 20%
+ 品类匹配 × 25%
+ 活跃信号 × 15%
+ 触达可行性 × 15%

等级规则：

80-100：A级
60-79：B级
40-59：C级
0-39：D级

客户列表中的每个客户都需要有：

综合商机指数
商机等级
评分拆解
推荐理由

十二、验收标准

生成结果必须满足：

1. 项目可以本地运行。
2. 页面结构完整。
3. 主要页面可以切换。
4. 搜索、筛选、收藏、详情、话术生成可交互。
5. 使用Mock数据，不依赖真实接口。
6. UI具备B2B SaaS质感。
7. 代码结构清晰，便于后续接后端。
8. 商机评分和推荐理由必须展示。
9. 客户详情页信息完整。
10. AI话术生成页可以模拟生成不同类型话术。
