AI商机雷达 PRD

版本：V1.0 产品名称：AI商机雷达 英文名：Opportunity AI 产品阶段：MVP V1 目标用户：餐饮供应链B2B销售 产品形态：Web端数据产品 + AI销售辅助工具

一、产品背景

1.1 行业背景

餐饮供应链企业、食材供应商、冻品供应商、调味品供应商、酒水供应商等B2B销售团队，普遍存在客户开发效率低、客户筛选依赖经验、销售拜访随机性强的问题。

传统销售开发客户的方式主要是：

搜索客户
↓
人工筛选
↓
挨个联系
↓
凭经验判断客户价值
↓
低效率拜访

这种方式存在几个核心问题：

  1. 客户数据多，但销售不知道哪些客户更值得优先跟进。
  2. 商家经营情况、门店扩张、品类匹配等信息分散，人工判断成本高。
  3. 销售拜访缺少明确切入点，话术和方案高度依赖个人经验。
  4. 企业无法沉淀“什么客户更容易成交”的规律。
  5. 数据产品多停留在查询层，不能直接帮助销售提升成交效率。

因此，需要一个从“数据查询”升级为“客户发现与销售行动建议”的产品。

二、产品定位

2.1 产品一句话定位

AI商机雷达，是面向餐饮供应链B2B销售的AI客户发现系统，基于餐饮商家数据、品牌数据、门店数据和经营信号，自动识别高价值客户，并提供客户推荐、商机评分、销售切入建议和拜访话术。

2.2 产品核心价值

传统数据工具解决的是：

我想查谁，就查谁。

AI商机雷达解决的是：

我卖什么，系统告诉我应该先找谁、为什么找他、怎么切入。

2.3 与传统数据查询工具的区别

对比项 传统餐饮数据工具 AI商机雷达
核心能力 数据查询 客户发现
用户动作 用户主动搜索 系统主动推荐
输出结果 商家列表、品牌信息 商机客户、评分、推荐理由、销售建议
使用价值 看数据 找客户、促成交
用户依赖 强依赖人工判断 AI辅助判断优先级
商业价值 信息服务 销售增长服务

三、产品目标

3.1 业务目标

MVP阶段主要验证以下问题：

  1. 销售是否愿意通过AI商机雷达寻找客户。
  2. AI推荐客户是否比人工筛选客户更有价值。
  3. 商机评分和推荐理由是否能被销售理解和信任。
  4. AI生成的销售切入建议是否能辅助销售开展跟进。
  5. 产品是否具备后续商业化收费潜力。

3.2 产品目标

MVP V1重点完成：

输入销售产品
↓
推荐高价值客户
↓
展示推荐理由
↓
生成销售建议
↓
支持销售导出/收藏/跟进

3.3 不做什么

MVP V1暂不做以下内容：

  1. 不做复杂CRM系统。
  2. 不做自动外呼。
  3. 不做自动加微信。
  4. 不做自动发营销消息。
  5. 不做复杂机器学习训练平台。
  6. 不做投资机构、加盟商、SaaS服务商等多用户角色。
  7. 不做完整销售业绩管理。
  8. 不做成交预测的绝对承诺。

  5. 酒水饮料销售。
  6. 火锅食材销售。
  7. 烧烤食材销售。
  8. 半成品菜销售。

用户角色：B2B销售

项目 描述
工作目标 找到潜在客户并促成合作
日常动作 搜客户、打电话、加微信、拜访、报价

五、核心使用场景

适合开发的火锅、烤肉、自助餐、韩餐等客户排行榜。

销售可以查看：

  1. 客户名称。
  2. 商机评分。
  3. 推荐原因。
  4. 门店数量。
  5. 所在城市。
  6. 客单价。
  7. 经营状态。
  8. 销售切入建议。
  9. 电话/微信/拜访话术。

用户输入：

  6. 推荐理由。

  8. 拜访切入点。

5.4 场景四：销售需要快速生成话术

销售点击客户后，系统自动生成：

  1. 微信开场话术。
  2. 电话沟通话术。
  3. 上门拜访话术。
  4. 产品切入话术。
  5. 跟进提醒话术。

六、商机定义

6.1 什么是商机

本产品中的“商机”指：

具备明确采购场景、销售触达价值和潜在成交可能的餐饮商家或品牌客户。

6.2 商机类型

商机类型 定义 示例
新客开发商机 从未合作但具备采购潜力的客户 新开火锅品牌、区域连锁餐饮
增长型商机 近期扩张明显，未来采购量可能增加 近90天新增门店较多
品类匹配商机 与销售产品高度匹配 卖牛肉卷，推荐火锅/烤肉客户
区域开拓商机 某城市、某区域出现集中客户机会 杭州烧烤客户TOP100
高价值品牌商机 门店数、客单价、品牌规模较高 多城市连锁品牌

6.3 商机等级

等级 分数 定义 建议动作
A级 80-100 高价值、高匹配、优先跟进 立即联系/重点拜访
B级 60-79 有价值、有匹配，值得跟进 纳入销售开发名单
C级 40-59 有一定潜力，但优先级一般 批量触达或观察
D级 0-39 当前价值较低或匹配较弱 暂不优先跟进

3. 推荐理由

3. 品类匹配：主营火锅场景，与牛肉卷、冻品类产品匹配度较高。
4. 切入建议：建议从稳定供货、标准化食材和成本优化角度切入。

  3. 品类匹配分析。

十、商机评分模型

10.1 评分原则

MVP V1采用规则评分模型，不直接使用复杂机器学习模型。

评分模型要求：

  1. 可解释。
  2. 可配置。
  3. 可追溯。
  4. 可根据用户反馈持续优化。
  5. 评分结果必须展示组成维度，不能只给总分。

10.2 综合商机指数

商机指数
= 增长潜力 × 25%
+ 规模价值 × 20%
+ 品类匹配 × 25%
+ 活跃信号 × 15%
+ 触达可行性 × 15%

总分为0-100分。

10.3 指标说明

1. 增长潜力

衡量客户是否处于增长阶段。

参考字段：

  1. 近90天新增门店数。
  2. 近180天新增门店数。
  3. 门店增长率。
  4. 评论增长率。
  5. 品牌热度变化。

2. 规模价值

衡量客户采购价值和规模潜力。

参考字段：

  1. 门店数量。
  2. 覆盖城市数。
  3. 客单价。
  4. 品牌连锁化程度。
  5. 评论总量。

3. 品类匹配

衡量客户经营场景与销售产品是否匹配。

示例：

用户销售产品 高匹配餐饮类型
牛肉卷 火锅、烤肉、自助餐、韩餐
芝士碎 披萨、烘焙、西餐、茶饮
鸡翅中 烧烤、炸鸡、小吃、快餐
调味料 中餐、快餐、连锁餐饮
冻品 火锅、烧烤、快餐、团餐

4. 活跃信号

衡量客户近期是否有经营活跃度。

参考字段：

  1. 近30天评论数。
  2. 近30天评分变化。
  3. 近30天新增门店。
  4. 近30天品牌热度。
  5. 门店经营状态。

5. 触达可行性

衡量销售是否能有效联系和跟进。

参考字段：

  1. 是否有电话。
  2. 是否有地址。
  3. 是否正常营业。
  4. 是否存在疑似闭店。
  5. 是否具备门店定位信息。
  6. 联系信息完整度。

十一、AI能力设计

11.1 AI能力边界

AI在MVP中主要负责：

  1. 用户输入理解。
  2. 品类匹配解释。
  3. 客户画像总结。
  4. 推荐理由生成。
  5. 销售话术生成。
  6. 风险提示生成。

AI不负责：

  1. 编造客户事实。
  2. 直接修改商机评分。
  3. 自动承诺成交概率。
  4. 自动联系客户。
  5. 替代销售决策。

11.2 AI输出要求

AI生成内容必须遵循：

  1. 基于已有数据。
  2. 不编造未提供事实。
  3. 推荐理由必须可解释。
  4. 话术必须自然、克制、适合B2B销售。
  5. 输出内容需要标注“依据”和“建议”。
  6. 对数据不足的情况要明确提示。

十九、风险与应对

19.1 数据质量风险

风险：

门店状态不准、电话无效、数据延迟。

应对：

  1. 展示数据更新时间。
  2. 增加信息无效反馈。
  3. 对疑似闭店客户降低评分。
  4. 触达可行性单独计分。

19.2 推荐不准风险

风险：

销售认为推荐客户没有价值。

应对：

  1. 推荐理由必须可解释。
  2. 用户可以反馈“推荐有效/无效”。
  3. A/B测试不同评分权重。
  4. 优先从单一品类场景验证。

19.3 AI幻觉风险

风险：

AI编造客户经营情况或采购需求。

应对：

  1. AI只能基于已有字段生成内容。
  2. 数据不足时必须提示“暂无足够信息”。
  3. 推荐理由区分“数据依据”和“销售建议”。
  4. 关键字段不由AI生成，只由结构化数据提供。

19.4 用户不持续使用风险

风险：

用户试用后觉得只是换一种方式查数据。

应对：

  1. 首页突出“今日推荐”。
  2. 推荐结果必须比普通查询更省时间。
  3. 支持收藏和导出，进入真实销售动作。
  4. 增加跟进状态，让用户形成使用习惯。

19.5 商业化风险

风险：

用户愿意试用，但不愿付费。

应对：

  1. 明确节省销售筛选时间。
  2. 验证推荐客户成交率提升。
  3. 按席位、导出量、区域数据包收费。
  4. 面向销售团队而不是个人销售收费。

二十二、商业化方向

22.1 可能收费方式

收费方式 说明

22.2 商业化前提

产品必须先证明：

推荐客户质量 > 人工搜索客户质量

核心证明方式：

  1. 推荐客户被收藏率更高。
  2. 推荐客户被导出率更高。
  3. 推荐客户跟进反馈更好。
  4. 推荐客户成交率更高。
  5. 销售愿意持续使用。

二十三、最终产品愿景

AI商机雷达的长期目标不是做一个餐饮数据查询平台，而是成为餐饮B2B销售的AI销售大脑。

长期形态是：

商家数据
+
品牌增长数据
+
品类需求理解
+
销售行为反馈
+
成交结果回流
+
AI推荐模型

最终帮助销售持续回答三个问题：

我该找谁？
为什么找他？
怎么更容易成交？

二十四、总结

AI商机雷达MVP V1的核心不是做“大而全”的餐饮数据平台，而是聚焦一个明确场景：

餐饮供应链销售找客户。

第一版必须优先跑通：

产品输入
↓
客户推荐
↓
商机评分
↓
推荐理由
↓
销售话术
↓
收藏/导出/跟进

只要能够证明AI推荐客户比人工筛选更高效、更容易被销售接受，产品就具备继续迭代和商业化的基础。
