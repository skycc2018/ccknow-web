ShareLLM 项目开发计划

企业内部 LLM 能力共享平台：提供模型接入、Prompt 管理、知识库 RAG、Agent 编排等能力。 制定时间：2026-06-21 | 周期：12 周（3 个月）

一、项目目标

1.1 三维目标

维度 目标
技术目标 构建统一 LLM 网关，支持多模型切换、流式输出、成本监控
业务目标 让非技术团队（运营、产品、销售）零代码使用 AI 能力
组织目标 沉淀企业 Prompt 资产、知识库资产，避免重复建设

1.2 代码库

  • 项目路径：/Users/skycc/Documents/openclaw/siyang/modelhub/new-api-main
  • 技术栈：Go 1.22+ (Gin/GORM) + React 19 + SQLite/MySQL/PostgreSQL
  • 定位：基于 new-api 构建企业级 LLM 能力共享平台（多租户改造）

二、技术架构

2.1 三层架构

├───────────────────────────────────────────┐ │ 应用层：ChatUI / API / 嵌入组件 │ │ - 多租户隔离、权限管理、使用审计 │ ├───────────────────────────────────────────┤ │ 能力层：LLM Gateway / RAG / Agent │ │ - 模型路由、Prompt 模板、知识检索 │ │ - 工作流编排（LangChain / LlamaIndex） │ ├───────────────────────────────────────────┤ │ 基础层：模型服务 / 向量库 / 对象存储 │ │ - 私有化模型（可选）、Milvus/PGVector │ │ - 成本监控、日志追踪 │ └───────────────────────────────────────────┘

2.2 技术栈选型

层级 选型 说明
前端 React + Ant Design / Next.js 快速构建管理后台
后端 Python (FastAPI) / Node.js LLM 生态 Python 更成熟
向量库 Milvus / PGVector 参考 milvus-learning-guide
模型网关 LiteLLM / 自研 统一多模型接口
工作流 LangChain / LlamaIndex Agent 编排基础
部署 Docker / K8s 参考现有 SaaS 部署经验

三、开发路线图

Phase 1：MVP（第 1-4 周）

功能 描述 优先级
统一 LLM 网关 支持 OpenAI / Claude / 国产模型（DeepSeek / 文心）统一接入 P0
对话界面 类 ChatGPT 的 Web UI，支持多轮对话、历史记录 P0
Prompt 模板库 基础模板管理（增删改查），支持变量注入 P0
用户认证 企业 SSO 登录（如飞书 / 钉钉），使用配额限制 P1

产出物：ShareLLM 网关设计、Prompt 模板库 PRD

Phase 2：RAG 知识库（第 5-8 周）

功能 描述 优先级
文档接入 支持 PDF / Word / 网页 / 飞书文档导入，自动分段 P0
向量检索 基于 Milvus / PGVector 的语义检索，支持混合搜索 P0
知识库管理 多知识库隔离、权限控制、更新策略（增量/全量） P1
引用溯源 答案附带来源段落，支持点击跳转原文 P1

产出物：ShareLLM RAG 架构、知识库管理设计

Phase 3：Agent 与自动化（第 9-12 周）

功能 描述 优先级
Agent 编排 可视化工作流设计（如 Dify / Coze 模式），支持工具调用 P1
API 开放 将 Agent / RAG 能力封装为 REST API，供业务系统调用 P1
使用分析 Token 消耗、成本分摊、热门 Prompt / 知识库统计 P2
插件市场 内部工具插件（如连接 ERP / CRM / 飞书） P2

产出物：ShareLLM Agent 设计、API 开放平台

五、风险与应对

风险 应对策略
模型调用成本失控 接入成本监控 + 配额管理 + 模型降级策略（GPT-4 → GPT-3.5）
知识库数据质量差 建立文档预处理规范（OCR 纠错、分段策略优化）
用户采用率低 与业务场景深度绑定（如直接嵌入 [[产品周会_核心要点_待办_机会点_20260526]] 纪要生成）
模型幻觉风险 接入幻觉检测、人工审核回环、答案置信度标记
数据泄露风险 私有化部署支持、敏感数据脱敏处理、访问日志审计

六、里程碑与验收标准

阶段 时间 里程碑 验收标准
MVP 1-4 周 内部线上 支持 3+ 模型，10+ 用户活跃使用
RAG 5-8 周 知识库上线 导入 5+ 业务文档，检索准确率 > 80%
Agent 9-12 周 对外服务 API 调用量 > 1000/日，覆盖 2+ 业务场景
