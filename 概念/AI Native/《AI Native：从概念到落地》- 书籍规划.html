《AI Native：从概念到落地》

一本关于 AI Native 思维方式与工程实践的系统性指南

目标读者：产品经理、技术负责人、企业决策者 核心交付：读完能独立设计 AI Native 产品，而非简单加 AI 功能

书籍结构

Part 1: 为什么 AI Native？

第一章：AI Enhanced vs AI Native - 1.1 一个案例看懂区别：传统搜索 vs Perplexity - 1.2 三个关键特征：意图驱动、持续学习、自主行动 - 1.3 范式转移：从"人适应机器"到"机器适应人"

第二章：行业拐点 - 2.1 2023：ChatGPT 引爆的不仅是技术 - 2.2 从玩具到工具到伙伴：AI 产品的三个阶段 - 2.3 为什么现在必须理解 AI Native？

Part 2: AI Native 核心原理

第三章：Prompt 即接口 - 3.1 自然语言成为新的 API - 3.2 结构化 Prompt 设计（CRISPE 框架） - 3.3 Prompt 工程：从随意到专业

第四章：上下文即状态 - 4.1 短期记忆：对话窗口的管理 - 4.2 长期记忆：向量数据库与语义搜索 - 4.3 状态持久化：让 AI "记得"用户

第五章：生成即服务 - 5.1 AI 生成内容成为核心交付物 - 5.2 从"查询数据"到"生成答案" - 5.3 涌现能力：零样本/少样本学习

第六章：工具调用与 Agent - 6.1 AI 不仅能说话，还能办事 - 6.2 ReAct 模式：思考-行动-观察循环 - 6.3 Multi-Agent：模拟人类团队协作

Part 3: 设计方法论

第七章：从用户任务出发 - 7.1 不是"加 AI 功能"，是"重新定义任务" - 7.2 意图理解：用户真正想要什么？ - 7.3 设计原则：减少认知负荷，建立信任

第八章：人机协作流程 - 8.1 AI 辅助 vs AI 自主的连续谱 - 8.2 设计"随时人工接管"机制 - 8.3 渐进式引入：从增强到原生

第九章：处理不确定性 - 9.1 AI 幻觉：不可避免，但可管理 - 9.2 容错设计：置信度评分与人工确认 - 9.3 兜底策略：当 AI 失败时怎么办？

Part 4: 工程实战

第十章：RAG 系统构建 - 10.1 文档处理与向量化 - 10.2 检索策略：从基础到高级 - 10.3 生产级坑点：语义漂移、上下文溢出、成本失控

第十一章：Agent 开发 - 11.1 单 Agent 架构设计 - 11.2 Multi-Agent 协作模式 - 11.3 工具定义与调用规范

第十二章：性能优化 - 12.1 Token 优化与成本控制 - 12.2 延迟优化：流式输出、量化、批处理 - 12.3 模型级联：简单任务用小模型

第十三章：安全与合规 - 13.1 输入安全：提示词注入防护 - 13.2 输出安全：事实核查与内容过滤 - 13.3 国内合规：数据安全法、个保法、生成式 AI 管理办法

Part 5: 实战案例

第十四章：知识库产品 - 14.1 某科技公司 10万+ 文档问答系统 - 14.2 架构设计与核心代码 - 14.3 效果评估与迭代优化

第十五章：智能助手产品 - 15.1 电商客服自动化改造 - 15.2 Agent 设计与工具调用 - 15.3 从 30% 到 75% 的自动解决率

第十六章：内容创作产品 - 16.1 Notion AI 的产品形态进化 - 16.2 上下文感知与工作流集成 - 16.3 从"写文档"到"描述需求"

Part 6: 落地与进化

第十七章：评估体系 - 17.1 什么算"好"的 AI Native 产品？ - 17.2 北极星指标设计 - 17.3 A/B 测试与持续迭代

第十八章：团队能力建设 - 18.1 角色转变：从工程师到 AI 设计师 - 18.2 Prompt 工程师的培养 - 18.3 数据质量：新的核心竞争力

第十九章：未来三年 - 19.1 多模态融合：文本+语音+图像+视频 - 19.2 Agent 生态：专业 Agent 市场 - 19.3 边缘 AI：端侧实时推理 - 19.4 自主系统：从辅助到代理
