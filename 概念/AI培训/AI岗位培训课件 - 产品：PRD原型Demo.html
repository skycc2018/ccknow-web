AI岗位培训课件 - 产品：PRD→原型→Demo

课程目标

让产品经理掌握用AI加速PRD撰写、原型设计、Demo制作的全流程方法

一、产品AI应用场景概览

场景 AI能力 预期效果
PRD智能生成 基于需求描述自动生成PRD框架 撰写时间节省60%
用户调研分析 自动分析用户反馈，提取需求 分析效率提升5倍
原型快速生成 文本描述转原型图/流程图 原型时间从2天→2小时
竞品分析 自动抓取竞品功能，生成对比 实时掌握竞品动态
Demo视频制作 AI生成产品演示脚本/配音 演示材料制作提速

二、核心技能：Prompt Engineering

2.1 标准Prompt模板（产品专用）

【背景】
我是[公司名]的[高级产品经理]，负责[产品名称/领域]。
当前处于[需求分析/方案设计/评审准备]阶段。

【角色】
请你扮演一位资深产品专家，曾主导过[行业]头部产品设计。

【输入材料】
- 产品定位：[一句话描述]
- 目标用户：[用户画像]
- 核心场景：[使用场景]
- 竞品参考：[列出3个]

【任务】
请帮我[具体需求]。

【输出格式】
1. 核心结论
2. 详细方案
3. 风险与边界
4. 待确认问题
5. 下一步行动

【校验标准】
- 符合用户真实需求（非臆想）
- 技术可行性评估
- 商业价值量化

2.2 高频场景模板

场景1：PRD框架生成 - 背景：需撰写[功能模块]的PRD - 角色：资深产品专家（腾讯/阿里P8级别） - 输入： - 功能目标：[解决什么问题] - 用户故事：[作为...我想要...以便...] - 业务流程：[简述] - 涉及系统：[列出] - 任务：生成完整PRD框架 - 输出要求： 1. 版本记录 2. 背景与目标 3. 用户故事地图 4. 功能详述（含交互逻辑） 5. 数据需求 6. 埋点方案 7. 发布计划 8. 风险清单 - 格式：可直接填入细节的标准模板

场景2：用户反馈分析 - 背景：收集到[100条]用户反馈，需提炼需求 - 角色：用户研究专家 - 输入：[粘贴用户反馈] - 任务：结构化分析 - 输出： 1. 反馈分类（功能/体验/Bug/建议） 2. 高频问题Top 10 3. 需求优先级矩阵（用户价值×实现成本） 4. 可直接排期的需求（附方案简述） 5. 需进一步调研的开放问题

场景3：竞品功能对比 - 背景：评估竞品[新功能]，判断是否跟进 - 角色：竞品分析专家 - 输入： - 竞品功能描述：[粘贴] - 我司现状：[当前方案] - 用户反馈：[相关反馈] - 任务：生成分析报告 - 输出： 1. 功能拆解（用户价值/技术实现/商业模式） 2. 与我司战略匹配度 3. 跟进成本评估 4. 差异化机会 5. 建议动作（跟进/放弃/差异化）

三、实操案例

案例1：PRD撰写提效

背景： 产品经理小王每次写PRD需3-4天，且经常遗漏边界情况。

解决方案：

PRD框架生成Prompt

请为以下功能生成PRD框架：
功能：智能客服机器人
目标：降低人工客服压力，提升响应速度
用户：电商平台消费者
场景：咨询订单/退换货/产品问题

要求：
1. 包含完整PRD章节
2. 每个章节给出示例内容
3. 标注需业务方确认的关键点
4. 列出常见边界情况（10条以上）
5. 附交互流程图描述（供设计同事参考）

边界情况补充Prompt

基于以上PRD，请补充：
1. 异常流程（网络中断/系统超时/数据异常）
2. 权限边界（未登录/无权限/已过期）
3. 数据边界（空数据/超长文本/特殊字符）
4. 并发场景（多人同时操作/重复提交）
5. 兼容性（老版本/不同设备）

格式：表格（场景/触发条件/系统响应/用户提示）

效果： - PRD撰写时间：4天 → 1.5天 - 评审通过率：50% → 85% - 开发返工率：30% → 10%

案例2：用户调研自动化

背景： 产品改版前需做用户调研，但时间和预算有限。

解决方案：

调研方案设计Prompt

背景：计划改版[首页信息流]
角色：用户研究专家
目标：了解当前痛点，验证改版方向

请设计调研方案：
1. 调研方法（定性/定量/组合）
2. 样本要求（数量/筛选条件）
3. 访谈提纲（8-10个问题）
4. 问卷设计（15-20题，含逻辑跳转）
5. 数据分析框架
6. 时间安排
7. 预算估算

访谈分析Prompt

以下是[10位]用户的访谈记录：
[粘贴]

请分析：
1. 用户画像聚类（3-4类）
2. 核心痛点（Top 5）
3. 需求频次与强度
4. 改版接受度预测
5. 风险用户群体
6. 功能优先级建议

效果： - 调研周期：2周 → 1周 - 洞察深度：表面需求 → 深层动机 - 方案命中率：60% → 90%

案例3：原型到Demo快速产出

背景： 评审会需要演示Demo，但设计资源紧张，只有文字描述。

解决方案：

原型描述生成Prompt

请根据以下PRD描述，生成详细的原型说明：
[粘贴PRD功能描述]

要求：
1. 页面结构（模块/布局/层级）
2. 交互流程（点击→反馈→跳转）
3. 状态变化（默认/悬停/选中/禁用）
4. 异常状态（空/加载/错误）
5. 动效建议（过渡/反馈）
6. 标注设计注意点

格式：可直接给设计师的Brief

Demo脚本生成Prompt

请为以下功能生成演示脚本：
功能：[智能推荐]
时长：3分钟
受众：管理层评审

要求：
1. 开场（30秒，吸引注意）
2. 痛点共鸣（30秒）
3. 方案展示（90秒，核心流程）
4. 数据支撑（30秒）
5. 结尾号召（30秒）

附：
- 每页的旁白文案
- 关键数据标注
- 可能的问题及回答要点

效果： - Demo准备时间：3天 → 半天 - 评审通过率：40% → 80% - 资源依赖度：高 → 低

五、常见误区与避坑指南

误区1：AI生成的PRD直接给开发 → AI是框架，细节需产品经理补充，尤其是业务逻辑。

误区2：用AI代替用户调研 → AI可辅助分析，但真实用户声音不可替代，需结合使用。

误区3：过度追求AI生成的完整性 → PRD不是越长越好，精准 > 全面，聚焦核心价值。

误区4：忽视AI的幻觉风险 → AI可能"编造"功能或数据，关键信息需人工核实。
